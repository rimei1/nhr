EN（建材・建具）モジュール: 商談（en_zenekonsh レコードタイプ）と取引先を軸に、予実管理・帳票・仕入先を管理。 カスタムメタデータ EN_FiscalYear__mdt（会計年度）・EN_Report__mdt（大阪/東京設定）を参照して集計処理が動く。

ENモジュール Batch処理フロー

NHR_EN_SalesReportBatch1〜5 が連鎖実行 → FormCreation__c に帳票データを書き込み → NHR_EN_SalesReportCtrl（VFコントローラ）が Excel 出力。
SOQLは全て NHR_Dao.cls に集約。定数は NHR_Constant.cls の BATCHNAME_* を参照。
ST（スチール）モジュール: 鉄鋼業の受注・納品・経営ダッシュボードを管理。 @InvocableMethod で Flow から Batch を起動し、スナップショット → ダッシュボードへ段階的に集計する。
JU（BPO/KPI）モジュール: KGI・KPI管理と独自商談テーブルを保持。 JU_BPOGoal__mdt（カスタムメタデータ）で地域別の目標値を管理し、NHR_JU_BPOKPIKGIReportBatch が集計処理を実行する。
CS（顧客管理）モジュール: パーセプションマッピング分析を中心に顧客関係度を管理。 perception__c は Contact の MasterDetail（主従） 子オブジェクト。 NHR_CS_PerceptualMappingBatchSchedule で定期実行される。
